# Code of Conduct

## Our Standard

We want this project to be useful, rigorous, and welcoming.

Examples of behavior that help:

- being respectful in disagreement
- giving concrete, technical feedback
- assuming good intent while staying honest
- keeping discussions focused on the artifact and the user problem

Examples of unacceptable behavior:

- harassment or intimidation
- personal attacks
- bad-faith disruption
- publishing private information without consent

## Scope

This applies to project spaces, issues, pull requests, and discussions related to Semantic Authority.

## Enforcement

Project maintainers may remove, edit, or reject comments and contributions that violate this code of conduct.
